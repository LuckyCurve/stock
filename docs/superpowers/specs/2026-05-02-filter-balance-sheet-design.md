# 自定义筛选面板 + 资产负债结构图 设计文档

日期: 2026-05-02

## 概述

为股票筛选器仪表盘新增两个功能：
1. **自定义筛选面板** — 汇总排名 tab 上方可折叠的筛选条，支持 9 项指标实时过滤
2. **资产负债结构图** — 深度详情页新增堆叠柱状图，支持绝对值/归一化两种模式

---

## 功能 1：自定义筛选面板

### 位置与布局

汇总排名 tab，表格上方。可折叠，折叠时为一行标题行（筛选仍生效）。

展开状态：
```
┌─────────────────────────────────────────────────────┐
│ 🔧 自定义筛选                               重置    │
│                                                      │
│  Yrs≥[20]  CAGR%≥[  ]  EPS CAGR%≥[  ]  ROICmed≥[10]│
│  Recent5≥[  ]  MinROIC≥[  ]  <10y≤[  ]  NetC%≥[  ]  │
│  芒格筛选 [全部 ▼]                                    │
│                                                      │
│  ✅ 5/25 通过筛选            [收起 ▲]                 │
├─────────────────────────────────────────────────────┤
│ 📊 汇总排名                                          │
```

折叠状态：
```
┌─────────────────────────────────────────────────────┐
│ 🔧● 自定义筛选 (5/25)              [展开 ▼]         │
├─────────────────────────────────────────────────────┤
│ 📊 汇总排名                                          │
```

### 9 项筛选项

| 列 | 类型 | 筛选方式 | 默认值 |
|---|---|---|---|
| Yrs | 数值 | ≥ 输入框 | 空（不限制） |
| CAGR% | 数值 | ≥ 输入框 | 空 |
| EPS CAGR% | 数值 | ≥ 输入框 | 空 |
| ROICmed | 数值 | ≥ 输入框 | 空 |
| Recent5 | 数值 | ≥ 输入框 | 空 |
| MinROIC | 数值 | ≥ 输入框 | 空 |
| <10y | 数值 | ≤ 输入框 | 空 |
| NetC% | 数值 | ≥ 输入框 | 空 |
| ✅ 芒格 | 枚举 | 下拉选择（全部 / ≥3/5 / =5/5） | 全部 |

### 交互细节

- 面板默认**展开**
- 「收起 ▲」居底部右侧，和状态行同一行；折叠后底部显示「展开 ▼」
- 「重置」在右上角，和标题同行
- 数值输入框：空值 = 不限制，输入即时生效（debounce 300ms）
- 芒格筛选下拉：`全部` / `≥3/5` / `=5/5`
- 底部状态行：`✅ N/25 通过筛选`，实时更新
- 有活跃筛选时，标题旁显示小圆点徽章 `🔧●`，折叠状态标题后显示 `(N/25)`
- 筛选**仅影响汇总表行显示**，不影响左侧 ticker 列表和图表对比 tab
- 不满足条件的行直接隐藏（非灰显）

### 持久化

筛选条件保存到 localStorage（key: `stock-screener-filters`），下次打开自动恢复。

### 数据层

筛选逻辑纯前端 JS，基于已有的 `DATA.summary` 数组过滤，无需改动 `compute.py` 或 `build.py`。

## 功能 2：资产负债结构图

### 位置

深度详情 tab，在现有 2x2 四联图表下方、芒格筛选卡上方。

```
┌──────────────────────────────────────────────┐
│  [收入趋势] [净现金/总资产] [ROIC趋势] [EPS YoY%] │  ← 现有 2x2
├──────────────────────────────────────────────┤
│  📊 资本结构              [绝对值 | 归一化]   │  ← 新增
│  ┌────────────────────────────────────────┐   │
│  │  堆叠柱状图 / 归一化堆叠柱状图         │   │
│  └────────────────────────────────────────┘   │
│  ■ 权益  ■ 长期债  ■ 短期债  ■ 现金          │
├──────────────────────────────────────────────┤
│  🏛 芒格筛选                                  │  ← 现有
```

### 图表规格

- 宽度：撑满深度详情内容区
- 高度：250px
- 类型：ECharts 堆叠柱状图

### 4 层数据（从底到顶）

| 层 | 数据项 | 颜色 | 说明 |
|---|---|---|---|
| 权益 | Total Equity | #1a73e8 蓝 | 最底层 |
| 长期债 | + LT Debt | #fbbc04 黄 | |
| 短期债 | + ST Debt | #ea4335 红 | |
| 现金 | + Cash & Cash Equivalents | #34a853 绿 | 最顶层 |

### 绝对值模式

- Y 轴单位 $M（百万美元）
- 堆叠高度 = 权益 + 长期债 + 短期债（≈ 总投入资本方向）
- 现金层在最上方，直观体现"现金覆盖了多少债务"
- tooltip 显示各项绝对值

### 归一化模式

- Y 轴 0-100%
- 每根柱子的 4 层总和归一到 100%
- 便于跨年份看结构变化，不受规模增长干扰
- tooltip 同时显示百分比和绝对值

### 切换按钮

- 位置：图表标题右侧
- 样式：和现有 stmt-tab 风格一致（蓝底白字 = 选中，蓝字白底 = 未选中）
- 默认：绝对值模式
- 切换状态保存在组件内部，不持久化

### 数据层

需要从 `raw_data.balance` 提取 4 个时间序列：
- `Total Equity`
- `+ LT Debt`
- `+ ST Debt`
- `+ Cash & Cash Equivalents`

数据已存在于 `roic_cache/` YAML，`compute.py` 的 `compute_all()` 返回的 `raw_data` 中已包含，无需改动 Python 端。仅需在 `template.html` 的深度详情渲染函数中新增图表逻辑。

## 不在范围内

- 筛选条件 URL hash 同步（未来可扩展）
- 资产负债结构图在图表对比 tab 的多 ticker 叠加（单个 ticker 的结构图已足够，多 ticker 堆叠对比无意义）
- 筛选面板的滑块控件（输入框已足够，滑块增加复杂度但收益有限）
